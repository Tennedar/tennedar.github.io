近況報告
電子身份之類陸續激活中，在僅校內SNS上四處晃盪也有不少收穫。

基本大概扎完痛包了！之後會上傳圖片，個人感覺很滿意~Z也說很好（？）在收更多谷之後大概會有第二版方案出現吧。總而言之扎痛包一點都不難，大概的步驟是
選定扎板軟/硬→確定要放的谷→購買吧唧掛件色紙等保護套→根據作品風格和谷子風格購買谷美→粘貼背景紙→咕谷子→粘貼谷子→粘貼谷美

以r這次為例詳細記錄一下心路歷程：
選定扎板軟/硬——賣痛包送的是軟扎板，軟扎板可以通過用亞克力膠黏在包上的方式，克服軟扎板容易塌的缺點。————使用軟扎板
確定要放的谷————うみねこ色紙+細胞神曲吧唧、方卡+芙芙吧唧————風格：優雅復古
購買吧唧掛件色紙等保護套————下單了吧唧保護套，方卡自帶自封袋，為了保護色紙本該賣卡磚但是上次也說過卡磚貴得要命，最後賣了亞克力板自己裁了一下（不然太大）
根據作品風格和谷子風格購買谷美————下單
蝴蝶、玫瑰（原作元素）鐳射（符合優雅要求）貼紙
書信文字（符合復古要求+原作有書信要素）燙金（符合優雅要求）貼紙
塑料金邊黑底玫瑰（原作要素）
白色蕾絲珍珠花邊（痛包常用配件，且復古優雅）
書信舊紙樣式素材紙（痛包常用配件，且復古）
金屬金色蝴蝶（原作要素且優雅）
金屬四角邊框（因為有方形的色紙可以裝飾用）
粘貼背景紙————購買到手的背景紙都是基本為長方形、邊緣不平，所以思路是留1~2張完整的貼在中間，然後剩餘的把紙撕開去填充即可
咕谷子————先貼貼紙，塑料配件後放，原則是不可以擋臉，儘量簡潔
粘貼谷子→粘貼谷美————有手就行

    String designation;
    double salary;

    }

        age = empAge;
    }
    public void empDesignation(String empDesig){
        designation = empDesig;
    }

        salary = empSalary;

        System.out.println("年齡："+age);

前情提要

總而言之，原來的那些圖床不好用。
為什麼不直接傳到github page所在的倉庫呢？因為眾所周知github免費版每個倉庫只有1G，和原先r使用的博客平臺wordpress相比反而更小，因為wordpress是圖片空間1G，而github page是圖片+代碼總共1G。

官網：PicGo手冊，選擇適合自己電腦的版本下載（r選擇2.3.1版本-x64）；同時，由於手冊中說请确保你安装了 Node.js， 并且版本 >= 16。，就來安裝一下~node.js。
不過，從r安裝完到現在使用為止根本就沒有用到它，或許也可以不裝。
配置圖床、獲取token
